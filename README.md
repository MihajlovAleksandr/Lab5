# Git Flow Lab - абораторная работа по ветвлению в Git
📋 Описание проекта
Учебный проект, демонстрирующий практическое применение модели ветвления Git Flow. Проект создан для освоения современных практик контроля версий и командной разработки.

🎯 Цель работы
Освоить многоветочную модель разработки Git Flow на практике, реализовав полный жизненный цикл проекта с различными типами веток.

🌳 Модель ветвления Git Flow
Основные ветки:
main (master) — стабильная производственная версия, содержит только релизные коммиты

develop — основная ветка разработки, куда интегрируются готовые функции

Вспомогательные ветки:
feature/* — разработка новых функций (ветки от develop, обратно в develop)

release/* — подготовка релизов (ветки от develop, в main и develop)

hotfix/* — экстренные исправления (ветки от main, в main и develop)

🚀 Реализованный сценарий
📁 1. Инициализация проекта
bash

git checkout -b develop
git commit -m "Initialize develop branch"
git checkout main
